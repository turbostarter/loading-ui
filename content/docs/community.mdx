We’re glad you’re here. Use the channels that fit how you like to collaborate.

## Issues and feature requests

For **bugs** and **concrete feature ideas**, open an issue on the project repository. Include steps to reproduce, expected vs. actual behavior, and your environment when it helps narrow things down.

## Questions and ideas

For **how-to questions**, **show-and-tell**, or **open-ended discussion**, prefer GitHub Discussions (or the tool your maintainers use) so answers stay searchable for everyone.

## Code of conduct

Be respectful, assume good intent, and keep feedback focused on the work. Harassment and exclusionary behavior are not acceptable.

## Contributing code

If you want to propose changes, see [Contributing](/docs/contributing) for workflow expectations before you open a pull request.
